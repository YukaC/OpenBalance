# Accesibilidad (a11y) — OpenBalance

Sin Chromatic ni Playwright. La red de seguridad es **lint jsx-a11y** + **axe-core en smoke** + revisión manual en pantallas críticas.

## Automatizado

| Capa | Qué hace |
|------|----------|
| ESLint | `eslint-config-next` (`next/core-web-vitals`) incluye `eslint-plugin-jsx-a11y`. No desactivar esas reglas en `eslint.config.mjs`. |
| Tests | `src/lib/a11y-smoke.test.ts` importa `axe-core`. El `axe.run` sobre un fixture HTML se **skippea** si no hay DOM (no hay jsdom hoy). |
| Script | `node scripts/a11y-check.mjs` — recuerda el proceso manual. |

```bash
pnpm lint
pnpm test -- src/lib/a11y-smoke.test.ts
node scripts/a11y-check.mjs
```

## Checklist manual (antes de release que toque auth/forms)

1. **Foco:** Tab llega a todos los controles; Escape cierra modal/form; trap de foco en `TransactionForm` / `ConfirmDialog`.
2. **Labels:** inputs de Auth, PIN, onboarding y form de transacción tienen `<label>` o `aria-label`.
3. **Errores:** campos inválidos usan `aria-invalid` + `aria-describedby` al mensaje (`role="alert"`).
4. **Contraste:** texto sobre `--paper` / dark tokens legible (Lighthouse Accessibility ≥ 90 en Auth + Resumen).
5. **Calendario:** en `MonthJumpCalendar`, flechas mueven el día enfocado; Tab solo entra/sale de la grilla.
6. **Toast:** botón cerrar visible; hover/focus pausa el auto-dismiss (ventana de undo).
7. **Sync chip:** `role="status"` / `aria-live="polite"` anuncia cambios de estado.
